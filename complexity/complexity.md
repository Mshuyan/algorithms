# complexity

## 时间复杂度

+ 定义

  算法的时间复杂度，用来度量算法的运行时间，记作: T(n) = O(f(n))。它表示随着 输入大小n 的增大，算法执行需要的时间的增长速度可以用 f(n) 来描述。

+ 注意

  + 大O表示法强调的是时间复杂度的增长趋势，只取增长趋势最大的关系式即可

    > 如：O(n^2^+n^3^)可以直接写成O(n^3^)

  + 时间复杂度全部按最坏情况进行计算

+ 常见的算法时间复杂度由小到大依次为：

  Ο(1) ＜ Ο(logn) ＜ Ο(n) ＜ Ο(nlogn) ＜ Ο(n^2^) ＜ Ο(n^3^) ＜ … ＜ Ο(2^n^) ＜ Ο(n!)

## 空间复杂度

> 空间复杂度与时间复杂度类似，只不过空间复杂度关心的是占用内存随输入n变化的速度
